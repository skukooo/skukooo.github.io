线性表

每一个线性表实例都是一系列元素的有序集合。

线性表的ADT的描述：

抽象数据类型linearList
{
    IndexOf(x):找到给定元素的索引，如果找不到则返回-1
    get(index):找到给定索引对应的元素
    insert(index):在给定索引处插入元素，索引大于index的元素其索引+1
    erase(index):在给定索引处删除元素，索引大于index的元素其索引-1
    size():获取线性表的长度
    empty():线性表为空时返回True
    outPut():按序输出线性表元素
}

树

  • 节点n1到nk的路径定义为节点n1、n2……nk的一个序列

注：关于树的高度有两种定义，一种以节点级数进行定义，根节点为1级，其子节点为2级…………，树的高度即为最大级数，另一种以节点间的连线段数定义，树的高度即为最大连线段树，树的深度=树的高度

  • 前序遍历
  • 中序遍历
  • 后序遍历
  • 层次遍历
二叉查找树(搜索树)

在二叉树的基础上增加一个约束——左子树的关键字的值 < 父节点关键字的值 < 右子树关键字的值

竞赛树

eg：
1、n个进行比赛的选手组成竞赛树的叶子节点，两个选手一组组成一个节点的左子节点和右子节点，胜利者被记录在该组的父节点中，直到确定根节点中记录的比赛选手，比赛结束
2、从n个数中找出最大的数或最小的数

优先级队列
平衡二叉树：AVL和红-黑树

AVL树的特点：左子树和右子树的高度差为1

红黑树特征
1、根节点和所有外部节点都是黑色
2、根节点至外部节点的路径上没有两个连续的红色节点（指针）
3、根节点至外部节点的所有路径上黑色节点（指针）数目相同
注：指向黑色子节点的指针为黑指针，指向红色子节点的指针为红指针

注：C++的STL的map和multimap是通过红-黑树实现的

分裂树

B树
对于大型字典（无法将整个字典存于内存中，需要进行磁盘读写），B树优于AVL和红-黑树

图

概念：用线连接起来的节点的集合，带箭头的连线称为有向边，不带箭头的称为无向边
如果图的所有边都是有向边，则为有向图，所有边为无向边，则为无向图

进程间通信的两种机制
  • 共享内存

注：上述代码只允许缓冲区同时出现BufferSize-1个待消费项

  • 消息传递

    • 直接通信
      进行直接通信的进程需要明确指定接收者和发送者

      • 对称
        接收方和发送方均需明确指定对方
      • 非对称
        只需要发送方指定接收方，接收方不需要明确指定发送方

    • 间接通信（通过邮箱或端口）
      邮箱可以由进程拥有，也可以由操作系统拥有

    • 同步或异步地发送消息

      • 阻塞发送：发送者阻塞，直到接收者成功收到消息
      • 阻塞接收：接收者阻塞，知道有可用消息到达
    • 客户机/服务器之间的通信
      • 套接字
      • RPC（远程过程调用）
      • 管道
        • 普通管道（匿名管道）
          通过管道通信的进程间具有明确的父子关系，且消息只能单向传递，消息传递结束管道即消失
        • 命名管道
          通信进程间无明确父子关系，消息可以双向传递，可以长期存在，并且可以由多个进程共同使用
同步
  • 当两个进程并发操作同一数据，最终得出的结果与访问顺序相关，这种情况称为竞争条件
  • 临界区问题
    • 进程请求进入临界区所执行的代码段称为进入区
    • 进程修改公共变量执行的代码段称为临界区
    • 进程退出临界区执行的代码段称为退出区
  • 互斥锁
    • 进入临界区的进程可以通过acquire()获取锁，在退出临界区时释放锁
  • 信号量
    • 信号量的两个原子操作
      wait(S)             Signal(S)
{                   {
while(S<=0)        S++; 
;                 }
S--;
}
      • 信号量分二进制信号量和计数信号量，二进制信号量相当于互斥锁
      • 死锁：一组进程等待某个事件，而该事件必须由改组进程内的某个进程产生
