const修饰符与指针

const <Type> *ptr：指向const对象的指针，不能通过解引用修改所指向的对象，const类型对象的地址只能赋值给此种类型的指针
<Type> *const ptr：指针的值不能被修改，但可以通过解引用修改指针所指向的对象

引用与指针的区别

引用类型在声明的时候必须被初始化：引用一旦被定义就不能更改它所指向的对象了
引用的声明方式：<Type> &ref=val，对引用的所有操作都被直接应用到它所指向的对象上了
只有const引用才能被字面常量初始化：字面常量是不可寻址的，为了实现引用编译器会生成一个临时对象，引用实际指向的就是该临时对象

程序员转行
劝退言论

1、2002年下半年正式入行，至今2007年4月一直从事软件开发工作
2、另一个技术水平很高的老师（在外企写单片机的，30岁，月薪8000-1万），告诉我的是：迟早要转行，就像他现在来当培训教师一样，原因：太累
3、累，大家都知道就不说了。为什么没有希望呢？因为我发现一个公司真正勤勤恳垦的实干是挣不到大钱的。
真正能挣到大钱的公司完全都是靠老板的个人关系到什么程度，要想在某个行业里成为软件老大，要看你和这个行业里的政府主管部门的关系如何。我看到了太多软件和他们公司的产品，一个字“烂”。
可那赚钱的速度，呵呵。其实赚多少钱，都是老板的，我们打工的不就是拿个死工资吗，我们更多的人不是连个受剥削的机会都找不到吗?每次面试刚从大学出来的计算机的本科生，我真想对他说：你何必要选择这一行？每次面试那些工作经历比我长，年龄比我大程序员，看着他唯唯诺诺的目光，我就想：曾几何时我也像他这样，被人横眉冷对的面试多少次，以后我是不是还会像他这样，再去看人家的脸色啊？心寒啊！
看着同期毕业的同学，都转了行的，在自己的行业里都混的不错，大部分都比自己挣钱多，有干头；就是挣钱少的，他也干的轻松啊，最简单得到就是和自己项目接洽企业或政府的信息部主管或网管，懂的不多，轻轻松松，钱比我们的多，有问题老找我们，面对他犯的低级错误，我们还得笑呵呵。我心里确实不平衡啊。
4、我大学由于种种原因没毕业，最高学历是高中，呵呵。就学历而讲，能干到我目前这个水平我觉的可以了，是时候激流勇退了

AVL树的特点：左子树和右子树的高度差为1

红黑树特征
1、根节点和所有外部节点都是黑色
2、根节点至外部节点的路径上没有两个连续的红色节点（指针）
3、根节点至外部节点的所有路径上黑色节点（指针）数目相同
注：指向黑色子节点的指针为黑指针，指向红色子节点的指针为红指针

注：C++的STL的map和multimap是通过红-黑树实现的

进程间通信的两种机制
  • 共享内存

注：上述代码只允许缓冲区同时出现BufferSize-1个待消费项

  • 消息传递

    • 直接通信
      进行直接通信的进程需要明确指定接收者和发送者

      • 对称
        接收方和发送方均需明确指定对方
      • 非对称
        只需要发送方指定接收方，接收方不需要明确指定发送方

    • 间接通信（通过邮箱或端口）
      邮箱可以由进程拥有，也可以由操作系统拥有

    • 同步或异步地发送消息

      • 阻塞发送：发送者阻塞，直到接收者成功收到消息
      • 阻塞接收：接收者阻塞，知道有可用消息到达
    • 客户机/服务器之间的通信
      • 套接字
      • RPC（远程过程调用）
      • 管道
        • 普通管道（匿名管道）
          通过管道通信的进程间具有明确的父子关系，且消息只能单向传递，消息传递结束管道即消失
        • 命名管道
          通信进程间无明确父子关系，消息可以双向传递，可以长期存在，并且可以由多个进程共同使用
同步
  • 当两个进程并发操作同一数据，最终得出的结果与访问顺序相关，这种情况称为竞争条件
  • 临界区问题
    • 进程请求进入临界区所执行的代码段称为进入区
    • 进程修改公共变量执行的代码段称为临界区
    • 进程退出临界区执行的代码段称为退出区
  • 互斥锁
    • 进入临界区的进程可以通过acquire()获取锁，在退出临界区时释放锁
  • 信号量
    • 信号量的两个原子操作

S--;
}
      • 信号量分二进制信号量和计数信号量，二进制信号量相当于互斥锁
      • 死锁：一组进程等待某个事件，而该事件必须由改组进程内的某个进程产生
