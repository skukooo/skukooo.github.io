#CMake 最低版本号要求
cmake_minimum_required (VERSION 2.8)
#项目信息
project (<项目名称>)
#加入一个配置头文件，用于处理 CMake 对源码的设置
configure_file (
“${PROJECT_SOURCE_DIR}/config.h.in”
“${PROJECT_BINARY_DIR}/config.h”
)
#是否使用自己的 MathFunctions 库
option (USE_MYMATH
“Use provided math implementation” ON)
#是否加入 MathFunctions 库
if (USE_MYMATH)
include_directories (“${PROJECT_SOURCE_DIR}/math”)
add_subdirectory (math)
set (EXTRA_LIBS ${EXTRA_LIBS} MathFunctions)
endif (USE_MYMATH)
#指定生成目标
add_executable(<项目名称> <源文件名>)
#当有多个源文件时,查找当前目录所有源文件，并将文件名存入变量DIR_SRCS
aux_source_directory(. DIR_SRCS)
add_executable(<项目名称> ${DIR_SRCS})
#添加 math 子目录
add_subdirectory(math)
#指定生成目标
add_executable(<项目名称> main.cc[${DIR_SRCS}])
#添加链接库
target_link_libraries(<项目名称> MathFunctions[${EXTRA_LIBS}])

const修饰符与指针

const <Type> *ptr：指向const对象的指针，不能通过解引用修改所指向的对象，const类型对象的地址只能赋值给此种类型的指针，(const修饰的是<Type>)
<Type> *const ptr：指针的值不能被修改，但可以通过解引用修改指针所指向的对象(const修饰的是ptr)

引用与指针的区别

引用类型在声明的时候必须被初始化：引用一旦被定义就不能更改它所指向的对象了
引用的声明方式：<Type> &ref=val，对引用的所有操作都被直接应用到它所指向的对象上了
只有const引用才能被字面常量初始化：字面常量是不可寻址的，为了实现引用编译器会生成一个临时对象，引用实际指向的就是该临时对象
const引用可以用不同类型的对象初始化或不可寻址的值（如字面常量）
一个指向const对象的指针的引用：const <Type> * const &ptr

枚举类型

在算术表达式中，枚举成员会被隐式转换成算数类型

#define将一个名字设置为预处理变量，#ifdef和#ifndef检查预处理变量是否已定义

迭代器

有迭代器的类型都拥有能返回迭代器的成员，如begin()和end()，begin返回指向容器第一个元素的迭代器，end返回指向容器尾元素下一位的迭代器

边界处理

1、将数组作为参数传递给函数时，函数访问数组元素不能超界
解决方案一：begin()函数返回数组指向数组首元素的指针，end()函数返回指向数组尾元素下一位的指针，通过比较两个指针的值来确定是否达到数组边界
解决方案二：给函数传递数组以及数组大小两个参数

函数

int main(int argc,char * argv[]),第二个形参是一个数组，第一个形参是数组中字符串的数量
函数的类型由函数的返回值类型与形参类型共同决定的

把函数当实参使用时，函数会被自动转换成指向该函数的指针，函数不能作为返回值，但函数指针可以。
返回指向函数的指针：int (*f1(int))(int*,int);

Javascript的几个重要更新节点
  • 1997：发布ES1
  • 1999：发布ES3，添加正则表达式和try/catch
  • 2009：发布ES5，添加strict mode（严格模式）和JSON支持
  • 2015：发布ES6，添加类和模块
  • 2016：发布ES7，添加指数运算符（**）和Array.prototype.includes
id选择器和class选择器

id选择器用于控制单一元素，class选择器用于控制一组元素

JavaScript的一些注意点

1、在函数内部，通过var关键字可以显式地声明一个变量为局部变量，不加var关键字的变量被认为是全局变量，会覆盖函数外的同名变量
2、执行流进入函数，该函数的执行环境会被推入执行环境栈，函数执行完成后被弹出栈，每个执行环境都有一个与之关联的变量对象，环境中的所有变量和函数都保存在这个对象中
3、子执行环境可以访问父执行环境中的变量，父执行环境不能访问子执行环境中的变量
4、标识符解析是一个从当前执行环境开始逐级向父级执行环境搜索的过程
5、与C、C++等语言以代码块划分作用域不同，JavaScript以执行环境划分作用域

红黑树特征
1、根节点和所有外部节点都是黑色
2、根节点至外部节点的路径上没有两个连续的红色节点（指针）
3、根节点至外部节点的所有路径上黑色节点（指针）数目相同
注：指向黑色子节点的指针为黑指针，指向红色子节点的指针为红指针

注：C++的STL的map和multimap是通过红-黑树实现的

进程间通信的两种机制
  • 共享内存

注：上述代码只允许缓冲区同时出现BufferSize-1个待消费项

  • 消息传递

    • 直接通信
      进行直接通信的进程需要明确指定接收者和发送者

      • 对称
        接收方和发送方均需明确指定对方
      • 非对称
        只需要发送方指定接收方，接收方不需要明确指定发送方

    • 间接通信（通过邮箱或端口）
      邮箱可以由进程拥有，也可以由操作系统拥有

    • 同步或异步地发送消息

      • 阻塞发送：发送者阻塞，直到接收者成功收到消息
      • 阻塞接收：接收者阻塞，知道有可用消息到达
    • 客户机/服务器之间的通信
      • 套接字
      • RPC（远程过程调用）
      • 管道
        • 普通管道（匿名管道）
          通过管道通信的进程间具有明确的父子关系，且消息只能单向传递，消息传递结束管道即消失
        • 命名管道
          通信进程间无明确父子关系，消息可以双向传递，可以长期存在，并且可以由多个进程共同使用
同步
  • 当两个进程并发操作同一数据，最终得出的结果与访问顺序相关，这种情况称为竞争条件
  • 临界区问题
    • 进程请求进入临界区所执行的代码段称为进入区
    • 进程修改公共变量执行的代码段称为临界区
    • 进程退出临界区执行的代码段称为退出区
  • 互斥锁
    • 进入临界区的进程可以通过acquire()获取锁，在退出临界区时释放锁
  • 信号量
    • 信号量的两个原子操作
      wait(S)             Signal(S)
{                   {
while(S<=0)        S++; 
;                 }
S--;
}
      • 信号量分二进制信号量和计数信号量，二进制信号量相当于互斥锁
      • 死锁：一组进程等待某个事件，而该事件必须由改组进程内的某个进程产生
