Posted on 2020-12-26 Edited on 2021-01-02 Valine:
HTTP请求

请求报文由请求方法、URI、协议版本、首部字段和实体构成
响应报文由协议版本、状态码、原因短语、首部字段和实体构成
HTTP是无状态协议，为了保存状态需要配合使用cookie
GET方法
POST方法
PUT方法
HEAD获取响应报文首部
DELETE：删除文件
OPTION：询问支持的方法
TRACE：追踪路径
CONNET：用隧道协议连接代理
pipelining：可以同时发送多个请求，而不是发送一个请求，等待响应然后再发送下一个请求
cookie：客户端发送请求，服务端在响应报文中添加set-cookie字段，客户端在下次发送请求时会在请求报文中添加cookie
通常报文主体等于实体主体，当传输时编码，会使实体主体发生变化
编码分类：压缩编码、分块编码、
多部份对象集合（multipart）：一份报文实体可以包含多种类型（图片、文本、视频…………），multipart在报文首部的Content-Type中定义
断点续传：报文首部中的Range属性可以指定请求实体的范围，以实现断点续传
内容协商（Content Negotiation）：请求同一个页面的不同版本（比如英文版和中文版）

响应状态码

状态码类别：

状态码 类别 原因短语
1XX informational（信息性） 请求正在处理中
2XX Success（成功） 请求正常处理完成
3XX Redirection（重定向） 需要附加操作以完成请求
4XX Client Error 服务器无法处理请求
5XX Server Error 服务器处理请求出错

200（OK）：成功
204（No Content）：成功处理请求，但响应报文不包含实体
206（Partial Content）：返回指定内容
301（Move Permanently）：资源已获得新的URI
302（Found）：临时移动，允许POST改GET（不符合标准）
303（See Other）：用GET方法从另一个URI处获取结果
304（Not Modified）：客户端发送带条件的请求，服务端允许访问资源，但未满足条件，响应报文不包含主体
307（Temporary Redirect）：符合标准，不从POST改为GET
400（Bad Request）：请求报文语法错误
401（Unauthorized）：需要提供通过HTTP认证的认证信息
403（Forbidden）：禁止访问
404（Not Found）：未找到资源或不允许访问
500（Internal Server Error）：服务器故障或Web应用bug
503（Service Unavailable）：超负荷或停机维护而无法处理请求

Web服务器

一台物理主机上可以托管多个域名，即可以运行多个服务器，访问这台主机上的任何一个域名，DNS会将域名解析为同一个IP，
代理：将客户端的请求转发给服务器，并将服务器的响应转发给客户端
经过代理的请求或响应报文中会追加via首部信息，代理的两种基准分类①是否会使用缓存②是否会修改报文
缓存代理：将请求的资源从源服务器上缓存下来，下次对相同资源的请求直接从代理发出，当源服务器上的该资源更新，则代理服务器从源服务器上缓存更新后的资源
透明代理：不对报文做任何修改的代理
网关：接受客户端的请求并处理，网关能使通信线路上的服务器提供非HTTP协议服务
隧道：通过SSL等加密手段，确保客户端与服务端之间通信的安全性

const修饰符与指针

const <Type> *ptr：指向const对象的指针，不能通过解引用修改所指向的对象，const类型对象的地址只能赋值给此种类型的指针，(const修饰的是<Type>)
<Type> *const ptr：指针的值不能被修改，但可以通过解引用修改指针所指向的对象(const修饰的是ptr)

引用与指针的区别

引用类型在声明的时候必须被初始化：引用一旦被定义就不能更改它所指向的对象了
引用的声明方式：<Type> &ref=val，对引用的所有操作都被直接应用到它所指向的对象上了
只有const引用才能被字面常量初始化：字面常量是不可寻址的，为了实现引用编译器会生成一个临时对象，引用实际指向的就是该临时对象
const引用可以用不同类型的对象初始化或不可寻址的值（如字面常量）
一个指向const对象的指针的引用：const <Type> * const &ptr

枚举类型

在算术表达式中，枚举成员会被隐式转换成算数类型

#define将一个名字设置为预处理变量，#ifdef和#ifndef检查预处理变量是否已定义

迭代器

有迭代器的类型都拥有能返回迭代器的成员，如begin()和end()，begin返回指向容器第一个元素的迭代器，end返回指向容器尾元素下一位的迭代器

边界处理

1、将数组作为参数传递给函数时，函数访问数组元素不能超界
解决方案一：begin()函数返回数组指向数组首元素的指针，end()函数返回指向数组尾元素下一位的指针，通过比较两个指针的值来确定是否达到数组边界
解决方案二：给函数传递数组以及数组大小两个参数

函数

int main(int argc,char * argv[]),第二个形参是一个数组，第一个形参是数组中字符串的数量
函数的类型由函数的返回值类型与形参类型共同决定的

把函数当实参使用时，函数会被自动转换成指向该函数的指针，函数不能作为返回值，但函数指针可以。
返回指向函数的指针：int (*f1(int))(int*,int);

内联（inline）

内联函数是为了解决调用函数降低程序运行效率的问题
在类中定义的函数为内联函数

2、管理方面，送变电应酬很多，酒桌文化极为浓厚，不适应，适应了这么久，还是不适应，这与我倾向于静静地做技术的性格格格不入。

4、对于丝毫不感兴趣的事，遇到挫折容易消极，因为没有积极面对，然后解决问题的动力，而且做事效率低，因为要分担出很大一部分精力来对抗消极情绪，每天5点醒，完全是因为只有5点到7点这段时间可以学计算机技术，所以情绪才能平复一点。

JavaScript中的所有节点类型都继承自Node类型,每个节点都有一个nodeType属性

Node.ELEMENT_NODE == 1

JavaScript的一些注意点

1、在函数内部，通过var关键字可以显式地声明一个变量为局部变量，不加var关键字的变量被认为是全局变量，会覆盖函数外的同名变量
2、执行流进入函数，该函数的执行环境会被推入执行环境栈，函数执行完成后被弹出栈，每个执行环境都有一个与之关联的变量对象，环境中的所有变量和函数都保存在这个对象中
3、子执行环境可以访问父执行环境中的变量，父执行环境不能访问子执行环境中的变量
4、标识符解析是一个从当前执行环境开始逐级向父级执行环境搜索的过程
5、与C、C++等语言以代码块划分作用域不同，JavaScript以执行环境划分作用域
6、内部函数访问外部函数变量就会形成闭包
7、在创建函数阶段，会创建一个包含全局执行环境变量对象的作用域链，该作用域链被保存在[[Scop]]数组中,调用函数时会创建该函数的执行环境，复制[[Scope]]中的对象构建作用域链，被调用函数的变量对象（此时是活动对象）被push到作用域链最前端（栈顶）
