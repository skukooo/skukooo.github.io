HTTP请求

请求报文由请求方法、URI、协议版本、首部字段和实体构成
响应报文由协议版本、状态码、原因短语、首部字段和实体构成
HTTP是无状态协议，为了保存状态需要配合使用cookie
GET方法
POST方法
PUT方法
HEAD获取响应报文首部
DELETE：删除文件
OPTION：询问支持的方法
TRACE：追踪路径
CONNET：用隧道协议连接代理
pipelining：可以同时发送多个请求，而不是发送一个请求，等待响应然后再发送下一个请求
cookie：客户端发送请求，服务端在响应报文中添加set-cookie字段，客户端在下次发送请求时会在请求报文中添加cookie
通常报文主体等于实体主体，当传输时编码，会使实体主体发生变化
编码分类：压缩编码、分块编码、
多部份对象集合（multipart）：一份报文实体可以包含多种类型（图片、文本、视频…………），multipart在报文首部的Content-Type中定义
断点续传：报文首部中的Range属性可以指定请求实体的范围，以实现断点续传
内容协商（Content Negotiation）：请求同一个页面的不同版本（比如英文版和中文版）

响应状态码

状态码类别：
|状态码|类别|原因短语|
|—|—|—|
|1XX|informational（信息性）|请求正在处理中|
|2XX|Success（成功）|请求正常处理完成|
|3XX|Redirection（重定向）|需要附加操作以完成请求|
|4XX|Client Error|服务器无法处理请求|
|5XX|Server Error|服务器处理请求出错|
200（OK）：成功
204（No Content）：成功处理请求，但响应报文不包含实体
206（Partial Content）：返回指定内容
301（Move Permanently）：资源已获得新的URI
302（Found）：临时移动，允许POST改GET（不符合标准）
303（See Other）：用GET方法从另一个URI处获取结果
304（Not Modified）：客户端发送带条件的请求，服务端允许访问资源，但未满足条件，响应报文不包含主体
307（Temporary Redirect）：符合标准，不从POST改为GET
400（Bad Request）：请求报文语法错误
401（Unauthorized）：需要提供通过HTTP认证的认证信息
403（Forbidden）：禁止访问
404（Not Found）：未找到资源或不允许访问
500（Internal Server Error）：服务器故障或Web应用bug
503（Service Unavailable）：超负荷或停机维护而无法处理请求

Web服务器

一台物理主机上可以托管多个域名，即可以运行多个服务器，访问这台主机上的任何一个域名，DNS会将域名解析为同一个IP，
代理：将客户端的请求转发给服务器，并将服务器的响应转发给客户端
经过代理的请求或响应报文中会追加via首部信息，代理的两种基准分类①是否会使用缓存②是否会修改报文
缓存代理：将请求的资源从源服务器上缓存下来，下次对相同资源的请求直接从代理发出，当源服务器上的该资源更新，则代理服务器从源服务器上缓存更新后的资源
透明代理：不对报文做任何修改的代理
网关：接受客户端的请求并处理，网关能使通信线路上的服务器提供非HTTP协议服务
隧道：通过SSL等加密手段，确保客户端与服务端之间通信的安全性
